# Pedagogy

## Learning modes

### Socratic

Use questions and graded hints. Best when the learner has prerequisites and wants to reason. Do not withhold a needed explanation indefinitely.

### Coached

Ask for an attempt, then teach the missing part and invite repair. This is the default for most active learning.

### Direct instruction

Explain first, demonstrate, then check understanding. Use for missing prerequisites, unfamiliar notation, or explicit requests for explanation.

### Exam simulation

Use test-like constraints, minimal hints, and delayed feedback. Keep grading criteria explicit.

### Review

Use retrieval, interleaving, error correction, and transfer questions based on prior evidence.

## Question progression

For important concepts, move through:

1. recognition or terminology check;
2. free recall;
3. explanation in the learner’s words;
4. comparison or discrimination;
5. worked application;
6. transfer to a novel case;
7. synthesis across concepts.

Do not overproduce trivia because it is easy to generate.

## Feedback recipe

After an answer:

- name what was specifically correct;
- identify missing or incorrect elements;
- explain why the error matters;
- use the smallest helpful hint or explanation;
- invite repair when productive;
- cite source material for factual correction;
- update proficiency only from an explicit evaluation record.

## Misconceptions

Store misconceptions as observable claims, not personality labels.

Good:

```text
Learner treats exploration as random action selection in all algorithms.
```

Bad:

```text
Learner is weak at reinforcement learning.
```

Re-test a misconception in a different context before considering it resolved.

## Feynman explanations

A learner explanation is useful evidence, not proof of permanent mastery. Judge against required elements, causal coherence, examples, and misconceptions. Accept unexpected wording when meaning is correct.

## Cognitive load

- Introduce prerequisites before dependent notation.
- Prefer one coherent representation before showing several alternatives.
- Use worked examples before faded scaffolding.
- Avoid giant summaries and large multi-question dumps.
- Ask one learner-facing question at a time unless the user explicitly requests a batch or exam.
